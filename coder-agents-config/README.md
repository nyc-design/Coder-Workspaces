# coder-agents-config

Central configuration for Coder Agents (chatd's loop) that's pushed to the
running Coder deployment by the
[`update-coder-agents-config.yaml`](../.github/workflows/update-coder-agents-config.yaml)
GitHub Actions workflow on every commit that touches this directory.

## Files

- `system-prompt.txt` — content posted to
  `PUT /api/experimental/chats/config/system-prompt`. Defines workspace-creation
  policy (use the `project-workspace` template, agent mode, `{Name}-Agents`
  naming convention) and general behavior for the central agent loop.

## Workflow contract

The workflow expects two repository secrets, the same ones used by
`update-coder-templates.yaml`:

- `CODER_URL` — base URL of the Coder deployment (e.g. `https://coder.tapiavala.com`)
- `CODER_SESSION_TOKEN` — admin session token (`coder token create --scope all`)

Pushed via PUT against `/api/experimental/chats/config/system-prompt` with body
`{"system_prompt": "<content>"}`. Endpoint is experimental — if Coder graduates
this out of experimental, update the workflow URL.

## Notes

- This system prompt applies to every Coder Agents chat in the deployment, on
  top of any per-workspace `~/.coder/AGENTS.md` and project-level `AGENTS.md`.
- The per-workspace prompt is generated by `11-agent-prompts.sh` and lives in
  the workspace image — see `workspace-images/base-dev/system_prompt.txt` and
  per-image `system_prompt_extension.txt` files.
- Don't put workspace-image guidance in this file; put it in the workspace
  prompts instead. This file is for what the central loop should know and do
  *before* it has a workspace context.
